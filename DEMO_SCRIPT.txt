[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at how the agent works under the hood. 
First, the user's request starts at the START node and immediately goes into the security checkpoint.
If any prompt injections or inappropriate topics are detected, the request is immediately routed to the security event node to block the action and log the warning.
Once cleared, the request reaches our Curriculum Orchestrator.
The Orchestrator uses AgentTools to coordinate with two specialized sub-agents: the Syllabus Planner, which creates structural study plan milestones, and the Quiz Generator, which creates multiple-choice practice quizzes.
Both specialized sub-agents are connected to our local Model Context Protocol server, which exposes tools to search educational databases, calculate a sustainable study pace, and validate quiz answers.
Once a syllabus is created, the system pauses at the ask student approval node using a RequestInput, prompting the user for approval.
If the student requests changes, the process student approval node routes the feedback back into the orchestrator for refinement, creating a secure, interactive loop.
